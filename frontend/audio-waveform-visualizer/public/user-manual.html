Overview
What this application does and how to use this manual

The Audio Duplicate Detection application helps audiobook producers and audio engineers find and remove repeated passages from recorded audio. It transcribes your audio file using Whisper, identifies duplicate phrases, aligns cut points to natural silence boundaries, and assembles a clean output file — all through a browser interface.

Prerequisites
What you need before you start

Account
A registered account on audio.precisepouchtrack.com. Contact your administrator to create one.

Align Cut Points to Silence
Ensure cuts happen at natural pauses, not mid-word
💡
This step is strongly recommended. Without alignment, cuts may fall mid-syllable. Alignment scans ±500 ms around each boundary and snaps it to the nearest silence — resulting in cleaner, more professional output.

Detection Algorithms
Choosing the right algorithm for your recording
Algorithm Best For Notes
TF-IDF Cosine Short recordings, exact phrase repeats Fast. Uses ≥0.85 cosine similarity. Good for clean recordings.
Retry-Aware Windowed Standard audiobook recordings Rolling 150-segment window. Balances speed vs. accuracy.
Retry-Aware + PDF When PDF script is available Adds a PDF position gate to reduce false positives in long books. Recommended when PDF is uploaded.
Anchor Phrase Global Recordings with widely spaced repeats No window limit. Catches duplicates any distance apart.
Multi-Pass Highest Accuracy Complex or long recordings Runs all four passes and merges results. Slowest but most thorough. Best choice for audiobooks.
AI Detection When semantic similarity matters Calls Claude claude-3-5-sonnet-20241022 to detect semantically similar passages. Requires API credits. Uses the same assembly flow as other algorithms.

Tips & Troubleshooting
Common issues and how to resolve them
Issue
Transcription seems stuck
If the transcription progress has not changed for more than 20 minutes, refresh the page. The task continues running server-side and the UI will pick up where it left off. If the status shows Failed, try re-uploading the file.
Issue
Too many false positives detected
Switch to an algorithm that accepts a PDF: Retry-Aware + PDF or Multi-Pass. If no PDF is available, increase the similarity threshold in the advanced settings. You can also manually deselect individual occurrences before assembling.
Issue
Assembly fails or produces a corrupt file
This can happen if the audio file has unusual formatting. Try converting to standard 44.1 kHz 16-bit WAV using Audacity before uploading. Re-run detection and assembly after re-uploading.
Issue
Page state lost after browser refresh
Detection results are saved to both the server and the browser's local storage. Refreshing the page will restore them. If results are not restored, re-run detection — the previous results are cleared when a new detection run starts.
Issue
AI detection returns no groups
The AI model may not have found duplicates above the similarity threshold. Try lowering the Similarity Threshold slider or switching to a dictionary-based algorithm instead.
💡
Best practice: For audiobook chapter files, use Multi-Pass + PDF with the similarity threshold at the default 0.85. Run Align to Silence before every assembly. Download the output and listen to the first few transitions before distributing.
